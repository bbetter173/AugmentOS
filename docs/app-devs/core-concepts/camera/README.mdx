The Camera Module provides photo capture and video streaming for MentraOS apps.

## Available Features

### [Photo Capture](./photo-capture.md)
Take high-quality photos from smart glasses with options for gallery saving and raw buffer access.

### [Video Streaming](./streaming.md)
Stream live video from smart glasses with two modes:

- **Managed Streaming** — Zero infrastructure. Cloud provides HLS/DASH/WebRTC playback URLs. Multi-app support. Optional re-streaming to YouTube, Twitch, etc.
- **Unmanaged Streaming** — Full control. Stream to your own endpoint using SRT, RTMP/RTMPS, or WHIP. Exclusive camera access.

See the [Video Streaming guide](./streaming.md) for full details, protocol comparison, and API reference.

## Quick Start

### Taking a Photo
```typescript
const photo = await session.camera.requestPhoto({ saveToGallery: true });
console.log(`Photo captured: ${photo.mimeType}, ${photo.size} bytes`);
```

### Starting a Managed Stream
```typescript
const result = await session.camera.startManagedStream();
console.log('Share this URL with viewers:', result.hlsUrl);
```

### Starting an Unmanaged Stream
```typescript
// SRT
await session.camera.startStream({ streamUrl: 'srt://your-server.com:4201?streamid=key' });

// RTMP
await session.camera.startStream({ streamUrl: 'rtmp://your-server.com/live/key' });
```

## Important Notes

- **Permissions**: Camera access requires the `CAMERA` permission. See [Permissions Guide](/app-devs/core-concepts/permissions).
- **Hardware**: Only available on camera-equipped glasses (e.g., Mentra Live)
- **Battery**: Extended streaming drains battery quickly
- **Privacy**: Always notify users when camera is active
