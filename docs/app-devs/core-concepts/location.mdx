Use `session.location` to get the user's GPS coordinates. You can get the latest known location or subscribe to continuous updates as the user moves.

## Get Latest Location

Request the most recent known location:

```typescript
const location = await session.location.getLatestLocation({ accuracy: "high" });

if (location) {
  session.display.showTextWall(`${location.lat}, ${location.lng}`);
}
```

The `accuracy` parameter controls GPS precision:

| Value | Description |
|-------|-------------|
| `"high"` | Best available accuracy (uses more battery) |
| `"standard"` | Balanced accuracy and battery usage |

## Subscribe to Updates

Get continuous location updates as the user moves:

```typescript
const cleanup = session.location.onUpdate((data) => {
  session.display.showTextWall(`Location: ${data.lat}, ${data.lng}`);
});

// Stop receiving updates
cleanup();
```

## Location Data

Each location event provides:

```typescript
session.location.onUpdate((data) => {
  data.lat;    // Latitude (number)
  data.lng;    // Longitude (number)
});
```

## Common Patterns

### Show location on session start

```typescript
app.onSession(async (session) => {
  const location = await session.location.getLatestLocation({ accuracy: "high" });

  if (location) {
    session.display.showTextWall(`You're at ${location.lat.toFixed(4)}, ${location.lng.toFixed(4)}`);
  } else {
    session.display.showTextWall("Location not available");
  }
});
```

### Location-aware content

```typescript
session.location.onUpdate(async (data) => {
  const nearby = await fetchNearbyPlaces(data.lat, data.lng);
  session.display.showReferenceCard("Nearby", nearby.map(p => p.name).join("\n"));
});
```

### Timezone from coordinates

```typescript
import tzlookup from "tz-lookup";

session.location.onUpdate((data) => {
  const timezone = tzlookup(data.lat, data.lng);
  session.logger.info(`User timezone: ${timezone}`);
});
```

## Permissions

Your app needs the **location** permission to access GPS data. Add it in the [Developer Console](https://console.mentra.glass/apps) when creating or editing your app.

Without location permission, `getLatestLocation` returns `null` and `onUpdate` receives no events.

## Migrating from v2

```typescript
// v2
session.location.getLatestLocation({ accuracy: "high" });
session.location.subscribeToStream({ accuracy: "standard" }, (data) => { ... });
session.events.onLocationUpdate((data) => { ... });

// v3
session.location.getLatestLocation({ accuracy: "high" });
session.location.onUpdate((data) => { ... });
```

`getLatestLocation` is unchanged. Location subscriptions are now via `session.location.onUpdate()` which returns a cleanup function. See the [Migration Guide](/app-devs/migration/overview) for the full list of changes.
