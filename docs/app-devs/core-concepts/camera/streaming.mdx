# Video Streaming

MentraOS supports live video streaming from smart glasses using three protocols: **SRT**, **RTMP/RTMPS**, and **WHIP (WebRTC)**. There are two streaming modes: managed streaming (cloud-orchestrated) and unmanaged streaming (direct to your endpoint).

## Types of streaming

### Managed Streaming

Managed streaming delegates ingest and playback to the MentraOS cloud. The SDK requests a stream, and the cloud provisions a Cloudflare live input and returns viewer URLs (HLS/DASH, optional WebRTC). Multiple apps can consume the same managed stream concurrently.

- Requires internet connectivity
- Non-exclusive camera access (cooperative with other MentraOS apps)
- Playback URLs become usable when status is "active"
- Cloud handles protocol selection (currently SRT)

#### Reference

- Method: `session.camera.startManagedStream(options?: ManagedStreamOptions): Promise<ManagedStreamResult>`
- Stop: `session.camera.stopManagedStream(): Promise<void>`
- Status events: `session.camera.onManagedStreamStatus(handler)`
- Status stream type: `StreamType.MANAGED_STREAM_STATUS`

Types:

```typescript
interface ManagedStreamOptions {
  quality?: "720p" | "1080p";
  enableWebRTC?: boolean;
  restreamDestinations?: { url: string; name?: string }[]; // optional RTMP fan-out
}

interface ManagedStreamResult {
  hlsUrl: string;
  dashUrl: string;
  webrtcUrl?: string;
  previewUrl?: string;
  thumbnailUrl?: string;
  streamId: string;
}
```

Managed stream URLs:
- **hlsUrl**: HTTP Live Streaming (HLS) manifest. Broadest player support on web and mobile; recommended default for viewers.
- **dashUrl**: MPEG-DASH manifest. Alternative adaptive streaming format for DASH-capable players/environments.
- **webrtcUrl**: Low-latency playback endpoint (available when `enableWebRTC` is true). Use for near real-time viewing with compatible players.
- **previewUrl**: Hosted player page suitable for quick testing and simple embedding.
- **thumbnailUrl**: Static/periodic thumbnail image for previews and UI cards; not a video stream.

Usage:

```typescript
// Subscribe to status BEFORE starting
const unsubscribe = session.camera.onManagedStreamStatus((status) => {
  if (status.status === "active") {
    console.log("HLS:", status.hlsUrl);
    console.log("DASH:", status.dashUrl);
    if (status.webrtcUrl) console.log("WebRTC:", status.webrtcUrl);
  } else if (status.status === "error") {
    console.error("Managed stream error:", status.message);
  }
});

// Start managed stream
const urls = await session.camera.startManagedStream({
  quality: "720p",
  enableWebRTC: true,
});

// Note: urls are returned immediately, but viewers should connect only after
// a status event reports status === "active".

// Stop when done
await session.camera.stopManagedStream();

// Cleanup status listener
unsubscribe();
```

Status handling guidelines:
- Do not present or share URLs to viewers until a status event with `status === "active"` is received.
- Treat `error` as non-recoverable for the current attempt. You may retry by calling `startManagedStream()` again.

### Unmanaged Streaming

Unmanaged streaming sends the camera feed directly from the device to your ingest endpoint. You control ingest, transcoding, and distribution. The protocol is determined by the URL you provide.

- **SRT** (`srt://`): Low-latency, resilient to packet loss. Recommended for custom infrastructure.
- **RTMP/RTMPS** (`rtmp://`, `rtmps://`): Widely supported by streaming platforms (YouTube, Twitch, etc.).
- **WHIP** (`https://`): WebRTC-based ingest. Ultra-low latency.

Characteristics:
- Exclusive camera access while active
- You manage endpoint availability, retries, and distribution
- Works with local or remote servers

#### Reference

- Method: `session.camera.startStream(options: StreamOptions): Promise<void>`
- Stop: `session.camera.stopStream(): Promise<void>`
- Status events: `session.camera.onStreamStatus(handler)`
- Status stream type: `StreamType.STREAM_STATUS`

Types:

```typescript
interface StreamOptions {
  streamUrl: string; // srt://, rtmp://, rtmps://, or https:// (WHIP)
  video?: VideoConfig;
  audio?: AudioConfig;
  stream?: StreamConfig;
  sound?: boolean; // play start/stop sounds (default: true)
}

interface VideoConfig {
  width?: number;      // e.g., 1280
  height?: number;     // e.g., 720
  bitrate?: number;    // bits per second, e.g., 2000000
  frameRate?: number;  // e.g., 30
}

interface AudioConfig {
  bitrate?: number;          // e.g., 128000
  sampleRate?: number;       // e.g., 44100
  echoCancellation?: boolean;
  noiseSuppression?: boolean;
}

interface StreamConfig {
  durationLimit?: number; // max duration in seconds
}
```

Usage:

```typescript
const cleanup = session.camera.onStreamStatus((status) => {
  console.log("Stream status:", status.status);
  if (status.status === "error") {
    console.error(status.errorDetails);
  }
});

// SRT stream
await session.camera.startStream({
  streamUrl: "srt://your-server.com:4201?streamid=my-stream",
});

// RTMP stream
await session.camera.startStream({
  streamUrl: "rtmp://your-server.com/live/stream-key",
});

// WHIP (WebRTC) stream
await session.camera.startStream({
  streamUrl: "https://your-server.com/whip/endpoint",
});

// ... later
await session.camera.stopStream();
cleanup();
```

## When to use each

| | Managed | Unmanaged |
|---|---|---|
| **Use when** | You need turnkey ingest with HLS/DASH/WebRTC playback URLs, cooperative camera access, or cloud-managed resilience | You need to stream to a custom endpoint, require full control, or operate without the managed pipeline |
| **Protocol** | Cloud-selected (SRT) | Your choice: SRT, RTMP/RTMPS, or WHIP |
| **Infrastructure** | None required | You provide the ingest server |
| **Camera access** | Non-exclusive (multi-app) | Exclusive |
| **Internet** | Required | Depends on your server (can be local network) |

## Re-streaming to social platforms

Managed streaming can fan out to RTMP destinations you specify. Provide `restreamDestinations` with platform ingest URLs (including stream keys).

```typescript
const result = await session.camera.startManagedStream({
  restreamDestinations: [
    { url: "rtmp://a.rtmp.youtube.com/live2/YOUR_KEY", name: "YouTube" },
    { url: "rtmp://live.twitch.tv/app/YOUR_KEY", name: "Twitch" },
  ],
});
```

## Checking for existing streams

Use `session.camera.checkExistingStream()` to detect if any stream is already active for the current user (managed or unmanaged). Useful for reconnection, avoiding duplicate streams, and coordinating across multiple apps.

```typescript
async checkExistingStream(): Promise<{
  hasActiveStream: boolean;
  streamInfo?: {
    type: "managed" | "unmanaged";
    streamId: string;
    status: string;
    createdAt: Date;
    // Managed streams
    hlsUrl?: string;
    dashUrl?: string;
    webrtcUrl?: string;
    previewUrl?: string;
    thumbnailUrl?: string;
    activeViewers?: number;
    // Unmanaged streams
    streamUrl?: string;
    requestingAppId?: string;
  };
}>
```

Example:

```typescript
const result = await session.camera.checkExistingStream();

if (!result.hasActiveStream) {
  await session.camera.startManagedStream();
  return;
}

if (result.streamInfo?.type === "managed") {
  console.log("HLS:", result.streamInfo.hlsUrl);
} else {
  console.log("Active stream:", result.streamInfo?.streamUrl);
  console.log("Requesting app:", result.streamInfo?.requestingAppId);
}
```

Notes:
- Managed streams support multiple apps; calling `startManagedStream()` will join the existing managed stream.
- Unmanaged streams are exclusive and block other streams until stopped.

## Permissions

All streaming requires the `CAMERA` permission in your app configuration. See [Permissions Guide](/app-devs/core-concepts/permissions).

## Operational notes

- Only one unmanaged stream can run at a time per device and it blocks managed streams while active.
- Managed streams allow multiple apps to consume the same device feed.

## FAQ

**How do I know when managed stream URLs are usable?**
Subscribe to `onManagedStreamStatus` and wait for `status === "active"` before sharing or embedding.

**Can I re-stream a managed stream to RTMP destinations like YouTube or Twitch?**
Yes. Provide `restreamDestinations` in `ManagedStreamOptions`.

**How do I check for an existing active stream after an app restart?**
Call `session.camera.checkExistingStream()` and inspect `streamInfo`.

**Does managed streaming require internet access?**
Yes. Unmanaged can work on local networks depending on your server reachability.

**Can I enable low-latency viewing?**
Set `enableWebRTC: true` in `ManagedStreamOptions`. Use `webrtcUrl` for playback.

**Which protocol should I use for unmanaged streams?**
- **SRT** for low-latency, resilient streaming over unreliable networks.
- **RTMP** for compatibility with existing platforms (YouTube, Twitch, etc.).
- **WHIP** for ultra-low latency WebRTC ingest.
